## 📘 步骤一：安装基础开发工具

### 1.1 安装 Visual Studio Code (VS Code)

VS Code 是目前最流行的代码编辑器，免费且功能强大。

#### Windows 安装

```bash
# 1. 访问官网下载
https://code.visualstudio.com/

# 2. 下载 VS Code User Installer (推荐)
# 双击安装包，一路点击"下一步"完成安装

# 3. 验证安装
# 打开命令提示符(CMD)，输入：
code --version
```

#### macOS 安装

```bash
# 方法1: 使用 Homebrew (推荐)
brew install --cask visual-studio-code

# 方法2: 直接下载
# 访问 https://code.visualstudio.com/
# 下载 .dmg 文件，拖拽到应用程序文件夹

# 验证安装
code --version
```

#### 安装必要插件

打开 VS Code，按 `Ctrl+Shift+X` (Windows) 或 `Cmd+Shift+X` (macOS) 打开插件市场，搜索并安装：

### 4.3 创建核心配置文件

创建 `backend/app/core/config.py`：

```python
from typing import List
from pydantic_settings import BaseSettings
from functools import lru_cache


class Settings(BaseSettings):
    """应用配置"""

    # 应用信息
    APP_NAME: str = "企业微信教务系统"
    APP_ENV: str = "development"
    DEBUG: bool = True
    SECRET_KEY: str

    # 服务器配置
    HOST: str = "0.0.0.0"
    PORT: int = 8000

    # 数据库配置
    DATABASE_URL: str

    # Redis 配置
    REDIS_URL: str = "redis://localhost:6379/0"

    # 企业微信配置
    WEWORK_CORP_ID: str
    WEWORK_CORP_SECRET: str
    WEWORK_AGENT_ID: int

    # JWT 配置
    JWT_SECRET_KEY: str
    JWT_ALGORITHM: str = "HS256"
    JWT_EXPIRE_MINUTES: int = 10080  # 7天

    # CORS 配置
    ALLOWED_HOSTS: List[str] = [
        "http://localhost:3000",
        "http://localhost:5173",
        "http://127.0.0.1:3000",
        "http://127.0.0.1:5173",
    ]

    # 文件上传
    UPLOAD_DIR: str = "./uploads"
    MAX_UPLOAD_SIZE: int = 10485760  # 10MB

    class Config:
        env_file = ".env"
        case_sensitive = True


@lru_cache()
def get_settings() -> Settings:
    """获取配置单例"""
    return Settings()
```

### 4.4 创建最小化应用

创建 `backend/app/main.py`：

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from app.core.config import settings

# 创建 FastAPI 应用
app = FastAPI(
    title=settings.APP_NAME,
    description="基于企业微信的轻量级教务管理系统",
    version="1.0.0",
    debug=settings.DEBUG
)

# 配置 CORS
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.ALLOWED_HOSTS,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)


@app.get("/")
async def root():
    """根路径"""
    return {
        "message": f"欢迎使用{settings.APP_NAME}",
        "version": "1.0.0",
        "docs": "/docs",
        "status": "running"
    }


@app.get("/health")
async def health_check():
    """健康检查"""
    return {
        "status": "healthy",
        "app_name": settings.APP_NAME,
        "environment": settings.APP_ENV
    }


if __name__ == "__main__":
    import uvicorn

    uvicorn.run(
        "main:app",
        host=settings.HOST,
        port=settings.PORT,
        reload=settings.DEBUG
    )
```

### 5.2 生产环境：使用 PostgreSQL

使用 Docker 运行 PostgreSQL：

```bash
# 1. 创建 docker-compose.yml
```

在项目根目录创建 `docker-compose-dev.yml`：

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    container_name: edu_postgres_dev
    environment:
      POSTGRES_USER: edu
      POSTGRES_PASSWORD: edu123456
      POSTGRES_DB: edu_wework
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    container_name: edu_redis_dev
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
```

```bash
# 2. 启动数据库服务
docker-compose -f docker-compose-dev.yml up -d

# 3. 验证 PostgreSQL 运行
docker ps

# 4. 连接数据库测试
docker exec -it edu_postgres_dev psql -U edu -d edu_wework

# 在 psql 提示符下输入：
\conninfo  -- 查看连接信息
\l         -- 列出数据库
\du        -- 列出用户
\q         -- 退出

# 5. 修改 .env 文件
DATABASE_URL=postgresql://edu:edu123456@localhost:5432/edu_wework
```

### 5.3 配置数据库迁移

```bash
# 1. 初始化 Alembic
cd backend
alembic init alembic

# 2. 编辑 alembic.ini
# 找到 sqlalchemy.url 这行，修改为：

# 或使用环境变量：
sqlalchemy.url = ${DATABASE_URL}

# 3. 编辑 alembic/env.py
```

修改 `backend/alembic/env.py`：

```python
from logging.config import fileConfig
```

```python
"""环境测试脚本"""
import sys

from app.core.config import settings


def test_python_version():
    """测试 Python 版本"""
    version = sys.version_info
    assert version.major == 3, "需要 Python 3"
    assert version.minor >= 11, "需要 Python 3.11+"
    print(f"✓ Python 版本: {version.major}.{version.minor}.{version.micro}")


def test_config():
    """测试配置加载"""
    assert settings.APP_NAME, "配置文件未正确加载"
    print(f"✓ 应用名称: {settings.APP_NAME}")
    print(f"✓ 运行环境: {settings.APP_ENV}")
    print(f"✓ 调试模式: {settings.DEBUG}")


def test_database():
    """测试数据库连接"""
    from sqlalchemy import create_engine, text

    engine = create_engine(settings.DATABASE_URL)
    try:
        with engine.connect() as conn:
            result = conn.execute(text("SELECT 1"))
            assert result.scalar() == 1
        print(f"✓ 数据库连接成功")
    except Exception as e:
        print(f"✗ 数据库连接失败: {e}")
        sys.exit(1)


async def test_redis():
    """测试 Redis 连接"""
    try:
        import redis.asyncio as redis
        client = redis.from_url(settings.REDIS_URL)
        await client.ping()
        await client.close()
        print(f"✓ Redis 连接成功")
    except Exception as e:
        print(f"⚠ Redis 连接失败: {e}")
        print("  (Redis 为可选，不影响开发)")


async def main():
    """主测试函数"""
    print("\n=== 后端环境验证 ===\n")

    try:
        test_python_version()
        test_config()
        test_database()
        await test_redis()

        print("\n✓ 所有核心组件验证通过！")
        print("\n可以开始开发了 🚀\n")

    except AssertionError as e:
        print(f"\n✗ 验证失败: {e}\n")
        sys.exit(1)
    except Exception as e:
        print(f"\n✗ 发生错误: {e}\n")
        sys.exit(1)


if __name__ == "__main__":
    asyncio.run(main())
```

## 🐛 常见问题解决

### 问题1: Python 版本不对

```bash
# 症状：运行 python 显示旧版本

# 解决方案：
# 1. 检查系统 Python 版本
python --version

# 2. macOS 可能需要使用 python3
python3 --version

# 3. Windows 可能需要重启终端
# 或使用 Python Launcher
py -3.11 --version
```

### 问题2: pip 安装失败

```bash
# 症状：pip install 报错

# 解决方案：
# 1. 升级 pip
python -m pip install --upgrade pip

# 2. 清理缓存
pip cache purge

# 3. 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题3: 虚拟环境激活失败

```bash
# Windows: PowerShell 执行策略错误
# 解决方案：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# macOS: zsh: permission denied
# 解决方案：
chmod +x venv/bin/activate
```

### 问题4: npm 安装速度慢

```bash
# 解决方案：使用国内镜像
npm config set registry https://registry.npmmirror.com

# 验证
npm config get registry
```

### 问题5: Docker 启动失败

```bash
# macOS: Docker Desktop 无法启动
# 解决方案：
# 1. 检查系统版本兼容性
# 2. 重启 Docker Desktop
# 3. 检查虚拟化是否启用
```

### 问题6: 数据库连接失败

```bash
# 症状：连接 PostgreSQL 失败

# 解决方案：
# 1. 检查 Docker 容器是否运行
docker ps | grep postgres

# 2. 查看容器日志
docker logs edu_postgres_dev

# 3. 重启容器
docker-compose -f docker-compose-dev.yml restart postgres
```
