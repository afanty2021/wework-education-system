# 现代教务系统数据库设计

## 1. 数据库架构概述

### 1.1 设计原则

- **领域驱动设计**: 按业务领域拆分数据库
- **读写分离**: 主库写入，从库查询
- **分库分表**: 大表水平拆分，避免单表数据量过大
- **冷热分离**: 历史数据归档，保持热数据性能
- **缓存优先**: 热点数据缓存，减少数据库压力

### 1.2 数据库集群规划

```yaml
数据库集群:
  # 核心业务库
  edu_user:     # 用户权限库
  edu_member:   # 会员库
  edu_course:   # 课程库
  edu_order:    # 订单库
  edu_payment:  # 支付库
  edu_schedule: # 排课库

  # 支撑业务库
  edu_message:  # 消息库
  edu_file:     # 文件库
  edu_log:      # 日志库
  edu_config:   # 配置库

分片策略:
  member_follow:   # 按member_id哈希分片(16个分片)
  order_master:    # 按年分片
  payment_flow:    # 按月分片
  attendance_log:  # 按日分片
```
